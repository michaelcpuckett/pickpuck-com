Modes of Interaction

Web Browser

Visual-Pointer-Keyboard Context

Vocabulary Terms for
  • Graphical User Interface

    The document tree is rendered into a graphical representation of the current state and gets refreshed 60 times a second.

  • Mouse Input

    The page can be scrolled and interactive regions of the GUI can be activated with a pointing pointing device, controlled with the dominant hand and fingers.

  • Touch Input

    The page can be scrolled by swiping and interactive regions of the GUI can be activated by tapping with the tip of a finger.

  • Keyboard Input

    The page can be scrolled with the space bar and interactive regions of the GUI can be tabbed to and then activated with the enter key.

Audio-Keyboard and Tactile-only Context

Vocabulary Terms for
  • Textual User Interface (Speech-Synthesized)

    The Accessibility tree is rendered into a textual representation of the user interface. It reacts to state changes. It is often announced as speech, but can also be configured to be seen on-screen or converted to Braille.

  • Full Access Keyboard Navigation

    When a screen reader is turned on, both interactive and non-interactive items are navigable with the arrow keys. When an item is navigated to, it is announced and becomes the focused item to perform actions on.

  • Refreshable Braille Display

    Refreshable Braille displays are used to read Accessibility tree information instead of getting the same information through speech synthesis.

Visual-Vocal Context

Vocabulary Terms for
  • Voice Recognition Software

    Software that can accept voice commands to do things like scroll the page, click links and buttons, and enter text.

    Using both voice recognition software and a screen reader, users can read and navigate a web page without facing or being within reach of their device.

  • Annotated Graphical User Interface

    Voice Recognition Software may allow selection of items using their accessible label. Additionally, unlabelled or mislabeled items can be selected using an annoated visual overlay with numbers or letters for easy identifcation.

Service-Mediated

Increasingly, there are times when a user is in the middle of something, and wants to use their smart phone or ambient smart speaker setup without using a traditional web browser application to access content from the web.

Audio-Vocal or Audio-Vocal-Visual Context

Vocabulary Terms for
  • Digital Assistant

    Many smart assistants work in a question-and-answer or conversation-based format, often triggered by a word or phrase.

    Smart assistants parse queries and then return the best answer. These services will often draw from a massive knowledge graph of structured data for encyclopedic knowledge. Other types of queries may utilize web pages, by way of search engines, to provide answers.

    The service that is parsing the web page needs to identify content relevant to the query and ignore irrelevent navigation and footer content.

    Some smart assistants have a touch screen for visual display, but still rely largely on the audio interface and vocal commands. Smart homes can be set up to receive vocal input and visual results can appear on any screen connected to the network.

Parsing Differences

Vocabulary Terms for

  • Accessibility Tree

    A representation of the DOM that is derived from elements' roles and ARIA attributes. Screen readers interact with the Accessibility Tree.

  • Structured Data

    A representation of the DOM that is derived from elements' microdata. Search engines and digital assistants may find and reuse bits of information from web pages, so the information is available without navigating to the web page itself.

  • Document Outline

    A representation of the DOM that is derived from headings or sectioning content.

  • HTML Spec Compliance

    Because some elements are only valid inside certain other elements, markup should be written with an eye to the spec. A <fieldset> tag isn't valid inside a <button> tag, for example. Additionally, some elements have restrictions on which ARIA roles can be applied, potentially affecting the accessibility tree.
