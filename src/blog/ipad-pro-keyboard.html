Unlocking the Keyboard on the iPad Pro

I bought an iPad Pro in hopes of replacing my MacBook Pro. I wanted to be able to walk to a coffee shop with a very light device in my backpack and work on some projects in a place where I could really focus.

Being a web developer makes this a bit more challenging. For one, iOS restricts access to the filesystem, and I'm used to being able to open up a terminal and run command-line development tools directly on my local machine. But it's possible to log in to a cloud server from iOS, so you can have total access to a different computer. Another issue is that Apple doesn't provide a code editor for iOS and the App Store is very limited. Fortunately, there is one great paid app called GoCoEdit that works as a code editor with built-in SSH/FTP capabilities.

After setting up my cloud server and configuring GoCoEdit, I was ready to rock! Technically, anyway. In the coffee shop, this didn't work so well. Even with the keyboard attachment, I forgot how many interactions required me to touch the screen. Raising my arm so much kept disturbing the people around me. I felt too embarrassed to focus on my work.

What was the problem exactly? The iPad Pro can connect to the keyboard cover accessory or a bluetooth keyboard. By default, apps can define keyboard shortcuts and there are a few system-level shortcuts. The system shortcuts mimic those on macOS, such as Command+Tab to open the App Switcher. There is also Command+H to go to the Home screen. But you'll soon find that it's impossible to navigate the entire system with just the keyboard.

To open up more keyboard shortcuts, you'll have to completely opt-out of the touch interface that you're used to. You probably won't want to be in this mode all the time, so you can set it to quickly toggle on and off by triple pressing the home button. You can do this by selecting VoiceOver under General > Accessibility > Accessibility Shortcut.

After triple pressing, the first thing you'll notice is the "voice" part of VoiceOver. This feature is designed for blind people, so if you're not blind, you may want to mute your device for now. The idea is that you can use the keyboard to navigate the user interface, and the voice will announce where you are. For our purposes, we just need to navigate the interface.

Press the arrow keys to navigate around. To select an item, press spacebar with the VoiceOver modifier keys (usually Control+Option). There are lots of keyboard shortcuts you will need to know before being able to interact completely without the touch interface, so take a day to practice before setting off to the coffee shop!

Besides this navigation mode (Quick Nav) there is a separate mode for interacting with editable text. To toggle between Quick Nav and the editor mode, press the Left and Right arrow keys together, without the VoiceOver modifier keys. Quick Nav controls the virtual cursor, and the editor mode controls the text cursor.

The final mode is the Rotor. It controls what the Up/Down arrow keys do in Quick Nav. You will probably want to leave this set to Containers or Lines. But we can also use it to completely mute VoiceOver audio. You'll want to enable Volume in the Rotor Menu in the Accessibility system settings. Press the Up and Right or Down and Left arrow keys together, without the VoiceOver modifier keys, to switch between Rotor actions. Then after selecting Volume in the Rotor, you can press Arrow Down to set VoiceOver's volume to 0. The last change is to disable Audio Ducking under VoiceOver in Settings. Then you can play your normal audio without VoiceOver interfering.

I hope Apple will eventually open up more keyboard shortcuts without having to turn on VoiceOver. But for now, this should keep your mind at ease while doing your work in a crowded place.
